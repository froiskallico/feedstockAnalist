feedStockAnalist

Esse aplicativo foi desenvolvido para facilitar a análise de disponibilidade de matérias primas de uma ou mais Ordem de Produção.

Nele você poderá indicar uma ou mais ordens de produção e o sistema irá, automaticamente, analisar, para cada matéria prima que compõe cada um dos produtos desta(s) OP(s) suas disponibilidades futuras, indicando quais materiais podem vir a faltar ao longo do tempo de acordo com as demandas empenhadas, já registradas em sistema.

Como acessar

Para acessar o aplicativo use seu navegador favorito para ir até http://app.datateck.com.br/

Login/Cadastro

Cadastro

Se é a sua primeira vez utilizando o feedStockAnalist, você precisará se cadastrar. Para fazer o seu cadastro, na página inicial do aplicativo clique em Cadastre-se.

Na tela de cadastro você precisa preencher todos os campos do formulário para se cadastrar na plataforma.

Login

Caso você já tenha se cadastrado, na tela inicial, basta digitar seus dados de Usuário e Senha.

Tela Inicial

Na tela inicial do aplicativo serão exibidas as análises já feitas anteriormente pelo seu usuário.

Caso você não tenha solicitado nenhuma análise ainda, você verá a tela abaixo, informando que ainda não existem análises.

Para começar uma análise clique em Nova Análise no cabeçalho ou no botão verde Iniciar nova análise. Ao clicar em um desses links você será redirecionado para a tela de Criação de Nova Análise;

Nova Análise

Na tela abaixo você deverá informar o número de uma ou mais Ordem(ns) de Produção para a(s) qual(is) será feita a análise. Para mais de uma OP, informe seus números separados com vírgulas.

Após informar a(s) OP(s), clique no botão verde Iniciar nova análise.

  • Depois de clicar você pode fechar a página. O sistema irá fazer a análise em segundo plano e, quando estiver tudo pronto, vai lhe enviar um email informando que pode acessar a análise (com um link direto para o relatório da mesma).

  • O tempo de processamento irá variar, quanto mais matérias primas houverem dentro da(s) OP(s) em análise, maior será o tempo.

  • As análises são criadas e armazenadas no banco de dados com informações (Estoques, OCs, OPs) referentes ao momento em que foram geradas. Ou seja, se você criar uma análise hoje e abrir a mesma daqui 90 dias os dados estarão defasados, mas isso fará com que você possa guardar uma "foto" do estado atual.

  • Você poderá gerar quantas análises quiser de uma mesma OP, sem problemas!

Lista de Análises

As análises já existentes serão exibidas na tela principal do aplicativo:

  • Para abrir o relatório da análise, basta clicar no botão Abrir
  • Se você quiser descartar uma análise, utilize o botão com este símbolo:

Relatório da Análise

Os relatórios de análises exibirão uma lista com todas as matérias primas para as quais houverem sido identificadas possíveis faltas de material, dentro do horizonte de programação do item (Conforme parâmetros do MRP). Aquelas matérias primas para as quais não foram identificadas iminência de falta não serão exibidas no relatório.

Interface geral

Resumo do relatório

Nesta seção é exibido o resumo geral do relatório da análise.

Nele constarão:

  • O número da(s) Ordem(ns) de Produção;
  • A quantidade total itens de matéria prima que foram analisadas;
  • A quantidade de itens de matéria prima em que foram identificadas possíveis faltas;
  • O custo financeiro total das ações sugeridas. Composto pela somatória dos valores abaixo:
    • O valor total das Ordens de Compra sugeridas a serem colocadas para suprir as possíveis faltas;
    • O valor total refernte as OPs para as quais o aplicativo sugere alteração;
Exemplo:

O aplicativo sugere a inserção de:

  • Uma OC no valor total de R$ 500,00;
  • Outra OC no valor total de R$ 250,00.

Além disso, o aplicativo também verifica que para outro produto já existe uma OC pendente para a qual apenas é necessária uma antecipação.

  • Essa alteração de data corresponde a uma movimentação financeira equivalente a R$ 175,00.

Dessa forma:

| VALOR | Descrição | | :-: | :: | | + R$ 500,00 | OC 1 | | + R$ 250,00 | OC 2 | | + R$ 175,00 | Antecipação | | = R$ 925,00 | Custo Financeiro total |

Listagem de Itens

Nesta seção serão listados todos os itens para os quais foram identificadas possíveis faltas:

Cada linha da listagem corresponde a um item com falta iminente. Na linha, é exibido o CPD do item e seu Código do Fabricante.

  • As linhas exibidas em cor vermelha correspondem a itens críticos. Ao passar o mouse sobre eles será exibido um alerta;

    Item Crítico = Item com data de necessidade de aquisição inferior ao LeadTime do Fornecedor;

  • As demais linhas são items com falta mas não críticos;

Detalhamento dos Itens

Ao clicar nos itens, serão expandidos seus relatórios de detalhamento individual.

O layout dos relatórios segue o mesmo padrão:

  • Dados do item: CPD e Código do fabricante;

  • Quantidade faltante: Somatório das quantidades comprometidas em OPs que não serão atendidas por falta de material;

  • Ação sugerida: Comprar e/ou Antecipar OC - Caso hajam OCs postas no sistema, que possam ser antecipadas, o sistema sugerirá a Antecipação. Caso hajam OCs mas não sejam suficientes para suprir o total faltante, o sistema sugerirá Antecipar/Comprar. Caso não existam OCs pendentes, a sugestão será apenas a Compra;

    Quando a cor de fundo da ação sugerida Comprar for Vermelha, como na imagem abaixo, significa que a quantidade sugerida a ser comprada excede o Estoque Máximo.

  • Custo das ações: Somatório do custo financeiro das ações sugeridas, seguindo a mesma lógica deste exemplo acima. Neste caso, o sistema exibirá os valores referentes a Antecipações e Compras separadamente, exemplo:

  • Navegação: Esta é a seção onde pode-se navegar pelos demais detalhes da análise:

    • Timeline: Linha do tempo, similar ao MRP, exibindo Datas, Saldos Iniciais, Quantidades Pendentes em OCs, Total Comprometido (OPs pendentes) e os Saldos finais;

    • OPs Faltantes: Lista as Ordens de Produção para as quais o sistema identificou que faltarão a matéria prima em questão;

    • Ordemns de Compra Pendentes: Lista as Ordens de Compra pendentes, quando houverem;

    • OCs para Antecipar: Lista as OCs que foram identificadas como "Antecipáveis" para suprir a falta de material;

    • Comprar: Informa ao usuário:

      • A quantidade sugerida a ser comprada;
      • A data em que o sistema identificou em que haverá a primeira falta;
      • O custo total da quantidade a ser comprada;
      • O MOQ do item;

      Quando o sistema identificar que é necessária uma compra que exceda o Estoque Máximo, também serão exibidos:

    • Estoque máximo: A quantidade calculada para Estoque Máximo (Conforme parâmetros do MRP);

      O sistema calcula a quantidade a ser comprada respeitando o Estoque Máximo, conforme parâmetros do MRP

    • Quantidade excedente: A quantidade a ser comprada que excede o Estoque Máximo;

    • Valor da quantidade excedente: O valor financeiro relativo à quantidade a ser comprada que excede o Estoque Máximo.

Excluindo uma Análise

Atenção: Uma exclusão de análise não pode ser desfeita!

Ao clicar no botão de exclusão de uma análise você será redirecionado para uma página de confirmação. Por favor, somente confirme se você realmente deseja excluir essa análise para sempre pois, uma vez excluída, ela não poderá ser recuperada de nenhuma forma!
